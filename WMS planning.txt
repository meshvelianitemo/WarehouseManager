Things I'd set up immediately

These are worth doing before writing your first feature:

Clean Architecture solution structure
EF Core
SQL Server
Dependency Injection
FluentValidation
MediatR
Serilog
Global exception middleware
JWT authentication
AutoMapper or Mapster (I lean toward Mapster for new projects)
Repository pattern only if it genuinely adds value (don't force it everywhere)
Unit of Work if you find you need to coordinate multiple repositories in a transaction
Logging

I'd configure Serilog from day one.

Use:

Console
Seq
Rolling file logs

This will save you hours when debugging.

File Storage

Even if document uploads come later, define an abstraction now:

public interface IFileStorageService
{
    Task<string> UploadAsync(Stream stream, string fileName);
    Task DeleteAsync(string path);
}

Initially it can save files locally.

Later you can switch to:

Azure Blob Storage
Amazon S3
MinIO

without changing the application layer.

One thing I think will make this project stand out

Most portfolio WMS projects stop at CRUD.

I'd build yours around workflows.

Instead of:

Create Product
Edit Product

focus on business operations like:

Receive Shipment

Perform Cycle Count
